# portal-h5-lab

本仓库用于归档和展示多个独立开发的 HTML 页面。这些页面通常用于实验、原型验证或作为静态内容的分发入口。

## 目录结构

```
portal-h5-lab/
├── pages/           # 存放各个独立的 HTML 页面文件（或子目录）
├── assets/          # 公共资源文件（样式、脚本、图片等）
├── README.md        # 本文件
└── index.html       # （可选）仓库根目录下的入口页面
```

- `pages/`：每个页面以一个独立文件或子文件夹形式存放，命名清晰，便于索引。
- `assets/`：存放跨页面共享的静态资源，如通用 CSS、JavaScript 库、字体或图标文件。
- `index.html`：可选的仓库首页，用于展示页面列表或导航。

## 页面归档说明

所有归档的 HTML 页面均以独立、自包含为原则。每个页面应尽量将样式和脚本内联或引用 `assets/` 中的公共资源，以减少外部依赖。

页面命名建议：使用英文小写、连字符分隔，如 `example-page.html`。如有多个相关页面，可放入同一子目录。

## 维护说明

- 新增页面：在 `pages/` 下添加文件或目录，并确保页面可独立运行。
- 更新页面：直接修改对应文件，注意保持向后兼容。
- 删除页面：移除文件或目录，并检查是否有其他页面引用其资源。
- 资源管理：公共资源请放置于 `assets/` 下，避免重复。

## 使用方式

你可以通过以下方式查看或部署这些页面：

- 直接打开 HTML 文件（浏览器本地预览）。
- 使用任意静态服务器托管整个仓库（如 GitHub Pages、Vercel、Netlify 等）。

本仓库不针对特定域名或网站，所有页面均为通用归档用途。
